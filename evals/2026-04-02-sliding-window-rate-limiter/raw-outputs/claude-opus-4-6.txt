

# Production-Ready Sliding Window Rate Limiter for Express.js

## Strategy & Architecture

### Algorithm Choice: Sliding Window Log

I'll implement the **sliding window log** algorithm (not the approximation/counter hybrid). This tracks individual request timestamps within the window, giving exact counts. For high-throughput scenarios, I'll also note where the counter approximation would plug in, but the log approach is more correct and the requirements call for production-ready accuracy.

### Architecture Overview

```
┌─────────────────────────────────────────────────────┐
│                   Middleware Layer                    │
│  rateLimiter(config) → Express.RequestHandler        │
│  - Route matching & config resolution                │
│  - Header generation (429, Retry-After, X-RateLimit) │
│  - Key extraction (IP, user, custom)                 │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│              Core Algorithm Layer                     │
│  SlidingWindowRateLimiter                            │
│  - slidingWindowCheck(key, limit, windowMs) → Result │
│  - Delegates to storage adapter                      │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│              Storage Abstraction Layer                │
│  interface RateLimitStore                            │
│  ├── MemoryStore (default, with auto-cleanup)        │
│  └── RedisStore  (pluggable, uses Lua scripts)       │
└─────────────────────────────────────────────────────┘
```

### Key Design Decisions

1. **Sliding window log** stores timestamps of each request; on check, we prune expired entries and count remaining ones. This gives exact sliding window semantics.
2. **Storage interface** exposes a single atomic `hit()` method that returns current count and reset time — keeps atomicity concerns inside the store.
3. **Redis adapter** uses Lua scripting for atomic ZRANGEBYSCORE + ZREMRANGEBYSCORE + ZADD — no race conditions.
4. **Route matching** supports exact paths, glob patterns via `picomatch`, and a fallback default.
5. **Key extraction** is pluggable (default: IP-based) so users can rate-limit by user ID, API key, etc.
6. **Memory store** uses a `Map<string, number[]>` with a periodic sweep interval to prevent memory leaks from expired entries.

### memory.store.ts

```typescript
// src/stores/memory.store.ts

import { RateLimitStore, StoreHitResult, MemoryStoreOptions } from '../types';

/**
 * In-memory sliding window log store.
 *
 * Each key maps to a sorted array of timestamps (ms).
 * Periodic sweeping removes stale keys to prevent memory leaks.
 *
 * Thread-safe in Node.js single-threaded context.
 * NOT suitable for multi-process deployments without sticky sessions.
 */
export class MemoryStore implements RateLimitStore {
  private store: Map<string, number[]> = new Map();
  private sweepInterval: ReturnType<typeof setInterval> | null = null;
  private maxWindowMs: number = 0; // Track largest window for sweeping

  constructor(private options: MemoryStoreOptions = {}) {
    const sweepMs = options.sweepIntervalMs ?? 60_000;
    if (sweepMs > 0) {
      this.sweepInterval = setInterval(() => this.sweep(), sweepMs);
      // Don't prevent Node.js from exiting
      if (this.sweepInterval.unref) {
        this.sweepInterval.unref();
      }
    }
  }

  async hit(key: string, windowMs: number, now?: number): Promise<StoreHitResult> {
    const currentTime = now ?? Date.now();
    const windowStart = currentTime - windowMs;

    // Track max window for sweep efficiency
    if (windowMs > this.maxWindowMs) {
      this.maxWindowMs = windowMs;
    }

    let timestamps = this.store.get(key);

    if (!timestamps) {
      timestamps = [];
      this.store.set(key, timestamps);
    }

    // Prune expired entries (timestamps is sorted, so find cutoff index)
    const cutoffIndex = this.binarySearchCutoff(timestamps, windowStart);
    if (cutoffIndex > 0) {
      timestamps.splice(0, cutoffIndex);
    }

    // Add current request
    timestamps.push(currentTime);

    const oldestRequestMs = timestamps.length > 0 ? timestamps[0] : currentTime;

    return {
      totalHits: timestamps.length,
      oldestRequestMs,
      currentRequestMs: currentTime,
    };
  }

  async peek(key: string, windowMs: number, now?: number): Promise<number> {
    const currentTime = now ?? Date.now();
    const windowStart = currentTime - windowMs;
    const timestamps = this.store.get(key);

    if (!timestamps) return 0;

    const cutoffIndex = this.binarySearchCutoff(timestamps, windowStart);
    return timestamps.length - cutoffIndex;
  }

  async reset(key: string): Promise<void> {
    this.store.delete(key);
  }

  async shutdown(): Promise<void> {
    if (this.sweepInterval) {
      clearInterval(this.sweepInterval);
      this.sweepInterval = null;
    }
    this.store.clear();
  }

  /**
   * Binary search to find the index of the first element > windowStart.
   * All elements before this index are expired and should be pruned.
   */
  private binarySearchCutoff(timestamps: number[], windowStart: number): number {
    let low = 0;
    let high = timestamps.length;

    while (low < high) {
      const mid = (low + high) >>> 1;
      if (timestamps[mid] <= windowStart) {
        low = mid + 1;
      } else {
        high = mid;
